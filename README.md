Project Brief

SESS is a Python-based mini project developed to efficiently manage event scheduling and room allocation.
The system allows users to add rooms, create events, book rooms, and view bookings in an organized manner.
It uses JSON for storing data permanently.
The system prevents scheduling conflicts by checking room availability before booking.
It improves resource utilization and reduces manual errors.
It provides a simple command-line interface for easy interaction.
It demonstrates real-world application of Python in scheduling and management systems.

Modules Used in Project
1. Main Module (main.py)

This is the main execution file of the project.
It contains the menu-driven system.
It connects all functionalities like add room, add event, booking, etc.
It handles user input and controls program flow.

2. Event & Room Management Module

This module handles creation and storage of events and rooms.
It allows adding new rooms with capacity and features.
It allows creating events with date, time, and department details.
It ensures proper organization of data.

3. Booking Module

This module manages room booking functionality.
It checks room availability before booking.
It prevents time conflicts and double booking.
It stores booking records efficiently.

4. File Handling Module

This module handles data storage and retrieval.
It saves data in JSON format.
It loads existing data when the program starts.
It ensures data persistence.

5. Search & Report Module

This module provides additional functionalities.
It allows searching events using keywords.
It displays all events and bookings.
It generates basic reports like total rooms, events, and bookings.

Python Libraries Used
JSON

Used for storing and retrieving data permanently.
Lightweight and easy to use.
Helps maintain structured data.
